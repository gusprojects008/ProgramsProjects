METODOS DE COMUNICAÇÃO E PROTOCOLOS

O QUE SÃO PROTOCOLOS NAS REDES: PROTOCOLOS SÃO REGRAS E FORMAS DE COMUNICAÇÃO E TROCA DE INFORMAÇÕES ENTRE COMPUTADORES,
EXISTEM DIVERSOS PROTOCOLOS PARA COMUNICAÇÃO E TROCA DE INFORMAÇÕES ENTRE COMPUTADORES NA REDE
UNS MAIS SEGUROS OUTROS MENOS SEGUROS SÓ QUE MAIS RÁPIDOS ENTRE OUTRAS FORMAS DE COMUNICAÇÃO TODAS
POSSUEM SEUS DIREFERENTES PROS E CONTRAS LADOS BONS E RUINS. MAS PARA QUE ACONTEÇA UMA COMUNICAÇÃO
ATRAVES DE UM PROTOCOLO É NECESSARIO UM MÉTODO PARA QUE A COMUNICAÇÃO POSSA INICAR E OCORRER
EXISTEM VARIOS METODOS NAS REDES O PRINCIPAL É O METODO SOCKET OU TOMADA!

PROCOLOS DE COMUNICAÇÃO MAIS USADOS:

HTTP/HTTPS: PROTOCOLO DE TRANSFERENCIA DE HIPER TEXTO, USADO PARA TRANSFERIR CONTEUDO DE UM SERVIDOR PARA CLIENTE ATRAVES DO SITES, E TAMBEM PARA CLIENTES SE COMUNICAREM E TROCAREM
INFORMAÇÕES COM SERVIDORES OU USAR SERVIÇOES E FUNCIONALIDADES DAQUELE SERVIDOR, O SERVIDOR ESTÁ
GERALMENTE ASSOCIADO A UM NOME DE DOMINIO ATRAVES DO DNS(SISTEMA DE NOMEAMENTO DE DOMINIO) E PODEMOS
ACESSAR UM SERVIDOR QUE ESTÁ SE COMUNICANDO ATRAVES DO HTTP DE MANEIRA SIMPLES COLOCANDO O NOME DO SITE
QUE ESTÁ ASSOCIADO AO SERVIDOR NO NAVEGADOR WEB E ASSIM O DNS IRÁ CONSULTAR O NOME DO SITE POR
EXEMPLO: www.servidorExemplo.com E TRADUZIR ESSE NOME PARA ENDEREÇO REAL DA MAQUINA SERVIDOR
QUE É POR EXEMPLO: 192.168.0.116 E ASSIM PODERMOS ACESSAR ELE, E ATRAVES DO PROTOCOLO HTTP
O SERVIDOR ENTENDER O TIPO DE REQUISIÇÃO QUE ESTAMOS FAZENDO E RETORNA A RESPOSTA OU O CONTEUDO
DISPONIVEL NO SERVIDOR E O NAVEGADOR IRÁ INTERPRETAR E MOSTRAR DA MELHOR MANEIRA POSSIVEL 


O QUE SÃO SOCKETS: SOCKETS SÃO MÉTODOS DE COMUNICAÇÃO USADOS PARA LIGAR UM COMPUATADOR AO OUTRO
E ASSIM SE COMUNICAREM E TROCAREM INFORMAÇÕES ATRAVES DE UM PROTOCOLO, MAS COMO QUE OCORRE O METODO
SOCKET PARA TROCAR E ENVIAR INFORMAÇÕES ENTRE UM COMPUTADOR E OUTRO?
PARA UMA CONEXÃO SOCKET ACONTECER PRECISA-SE:

DE UM SERVIDOR E UM CLIENTE!

O SERVIDOR ESPERA(AGUARDA) INFORMAÇÕES OU CONEXÕES DEPENDENDO DO PROTOCOLO QUE A MAQUINA SERVIDOR
ESTEJA USANDO PARA SE COMUNICAR OU TROCAR INFORMAÇÕES COM OUTROS COMPUTADORES DE COMPUTADORES DE FORA, MAS PARA ISSO
É NECESSARIO QUE ELE TENHA UM ENDEREÇO IP PARA SER LOCALIZADO E QUE NELE TENHA UM PROGRAMA
QUE SEGUE UM PROTOCOLO PARA ATENDER AS INFORMAÇÕES DOS COMPUTADORES DE FORA(EXTERNOS) OU SEJA
CLIENTES, ESSE PROGRAMA IRÁ FUNCIONAR EM UMA PORTA ASSCOCIADA AO IP DA MAQUINA SERVIDOR PARA
QUE OS COMPUTADORES POSSAM INTERAGIR COM A MAQUINA SERVIDOR ATRAVES DOS PROTOCOLO E PROGRAMA
QUE ATENDA ELAS, E ESSE PROGRAMA POSSA FUNCIONAR EM UMA PORTA UNICA E EXCLUSIVA.

UMA MAQUINA SERVIDOR PODE TER DIVERSOS SERVIÇOS DEFERENTES QUE ATENDEM INFORMAÇÕES OU CONEXÕES DE COMPUTADORES
DE FORA DE FORMA DIFERENTE OU SEJA ATRAVES DE DIFERENTES PROTOCOLOS DE COMUNICAÇÃO MAS PARA CADA
SERVIÇO NA MAQUINA SERVIDOR QUE ATENDEM REQUISIÇÕES DE FORA ATRAVES DE UM PROTOCOLO DEVE HAVER UMA PORTA ESPECIFICA ASSOCIADA AQUELE SERVIÇO QUE SEJA  DIFERENTE DAS OUTRAS PORTAS ASSOCIADAS AOS OUTROS SERVIÇOS.
 

1 - O QUE É DNS: É UM SISTEMA USADO PARA ASSOCIAR NOMES FACEIS E LEGIVEIS PARA HUMANOS A ENDEREÇOS IP:

UM SITE COM O ENDEREÇO IP REAL É 10.0.10.9 MAS ATRAVES DO SISTEMA DE NOMEAMENTO DE DOMINIO(DNS)
ELE SE TORNA: www.siteExemplo.com ASSIM POSSIBILITANDO ACESSARMOS SITES MAIS FACILMENTE APENAS COLOCANDO OS NOMES ASSOCIADOS A ELES 

MAIS SOBRE PROTOCOLOS:

OS PROTOCOLOS SÃO A PEÇA CHAVE PARA A CONEXÃO COMUNICAÇÃO E TRASNFERENCIA DE ARQUIVOS  ENTRE O CLIENTE E O SERVIDOR OU ENTRE SERVIDOR E CLIENTE. 

SINAIS E INFORMAÇÕES QUE O PROTOCOLO HTTP/HTTPS USA PARA TROCAR INFORMAÇÕES COM O CLIENTE 
SOBRE A CONDIÇÃO DO SERVIDOR OU SOBRE A INTERAÇÃO DO CLIENTE COM O SERVIDOR:

STATUS DO SITE OU SISTEMA: OS CODIGOS QUE OS SITES DÃO: COMO 404, 505, 200 ENTRE VARIOS, SÃO USADOS PARA DIZER COMO ANDA O FUNCIONAMENTO DO SITE SE SUA REQUISIÇÃO DEU CERTO E SE A UM ERROR NO CLIENTE OU NO SERVIDOR E CADA NUMERAÇÃO TEM UM SIGNIFICADO PARA DIZER O TIPO DE ERRO OU SE FUNCIONOU COMO DEVERIA, ESSES NUMEROS ESTÃO CLASSIFICADOS EM 5 TIPOS:

DOS CODIGOS 100 ATÉ 199: É UMA RESPOSTA INFORMATIVA DO SITE PARA DIZER QUE ESTÁ DANDO CERTO A SOLICITAÇÃO OU ENVIO DO DADO

200 ATÉ 299: DEU CERTO A SOLICITAÇÃO OU ENVIO 

300 ATÉ 399: HOUVE UM REDIRECIONAMENTO, QUE VOCÊ FOI REDIRECIONADO A UMA OUTRA PAGINA APOS A SOLICITAÇÃO

400 ATÉ 499: QUANDO OCORRE ERROS DO CLIENTE, QUANDO A SOLICITAÇÃO É RECUSADA INVALIDA, QUANDO NÃO É BEM SUCEDIDA );

500 ATÉ 599: QUANDO OCORRE ERROS NO SERVIDOR, QUANDO O SERVIDOR NÃO ESTÁ FUNCIONANDO COMO DEVERIA E ESTÁ COM PROBLEMAS PARA RECEBER A SOLICITAÇÃO DO CLIENTE

EXISTEM VARIOS PROTOCOLOS USADOS PARA A COMUNICAÇÃO ENTRE CLIENTE E SERVIDOR, PARA TRASNFERIR ARQUIVOS E SE COMUNICAREM ENTRE SI COMO ESTES:

TCP/IP: O PRINCIPAL PROTOCOLO USADO PARA A COMUNICAÇÃO E ENVIO E RECEBIMENTO DE DADOS NA INTERNET, TCP SIGNIFICA PROTOCOLO DE CONTROLE DE TRANSMISSÃO E IP SIGNIFICA PROTOCOLO DE INTERNET E QUASE TODOS OS OUTROS SÃO BASEADOS NELE, ELE É BASEADO EM CONEXÃO E TROCA DE INFROMAÇÕES BIDIRECIONALMENTE E DE MANEIRA CONFIAVEL, O METODO DE COMUNICAÇÃO DESSE PROTOCOLO É O SOCKET ONDE UMA MAQUINA ESPERA POR CONEXÃO ATAVES DE UM SERVIÇO QUE ATENDE CONEXÃO DE UMA MAQUINA DE FORA, ESSE SERVIÇO É ASSOCIADO A UMA PORTA NO SERVIDOR E AS MAQUINAS DE FORA IRÃO SE CONECTAR NA MAQUINA SERVIDOR ATRAVES DO ENDEREÇO IP E PORTA DO SERVIDOR PARA SE COMUNICAREM COM ELE, AO CLIENTE SE CONECTAR E INICIAR UMA CONEXÃO SOCKET ATRAVES DO
PROTOCOLO DE COMUNICAÇÃO TCP/IP NO CLIENTE É CRIADO TAMBEM UM SOCKET USADO PARA RECEBER RESPOSTA OU INFORMAÇÕES DO SERVIDOR E TAMBEM PARA O SERVIDOR RECONHECER DE ONDE VEM A CONEXÃO OU A INFORMAÇÃO OU SOLICITAÇÃO, ESTE SOCKET CRIADO NO CLIENTE É CHAMADO SOCKET LOCAL E É CRIADO E ATRIBUIDO AUTOMATICAMENTE ATRAVES DO SISTEMA OPERACIONAL.

DHCP: PROTOCOLO DE CONFIGURAÇÃO DINAMICA DE ENDEREÇOS (HOST) EM INGLES FICA DYNAMIC HOST CONFIGURATION PROTOCOL, USADO PRINCIPALMENTE PELOS PROVEDORES DE INTERNET PARA FACILITAR A DISTRIBUIÇÃO DE IP PARA APARELHOS CONECTADOS A REDE PARA PODEREM SER IDENTIFICADOS E SE COMUNICAREM 

FTP: UM DOS PRIMEIROS PROTOCOLOS, FILE TRASNFER PROTOCOL PROTOCOLO DE TRASNFERENCIA DE ARQUIVOS ENTRE CLIENTE E SERVIDOR, EXISTE A VERSÃO DELE SEGURA A FTPS 

SSH: OUTRA FORMA DE COMUNICÇÃO E TRASNFERENCIA DE ARQUIVOS ENTRE CLIENTE E SERVIDOR MAS DE UMA FORMA MAIS SEGURA UTILIZANDO LOGIN COMO O NOME DE USUARIO DA MAQUINA QUE VOCÊ IRA SE CONECTAR, O IP DELA E A SENHA

SMB: FORMA DE COMUNICAÇÃO ENTRE MAQUINAS WINDOWS PARA TRASNFERENCIA DE ARQUIVOS  

SMTP: PROTOCOLO TRANSMISSÃO E RECEBIMENTO DE EMAIL

EXISTEM DIVERSOS OUTROS PROTOCOLOS DE REDE PARA COMUNICAÇÃO E TRASNFERENCIA DE ARQUIVOS ENTRE MAQUINAS CLIENTE E SERVIDOR.



ENTENDA MAIS COMO OS SITES FUNCIONAM E SEUS RECURSOS:

O QUE É UM HEADER NO HTTP:

É ONDE CONTEM INFORMAÇÕES SOBRE A SOLICITAÇÃO DO CLIENTE E A RESPOSTA DO SERVIDOR , INFORMAÇÕES UTEIS PARA MELHORAR A PRECISÃO DA REQUISIÇÃO DO CLIENTE E DA RESPOSTA DO SERVIDOR, NA REQUISIÇÃO DE UM CLIENTE PARA O SERVIDOR SÃO NOS CABEÇALHOS QUE ARMAZENAM INFORMÇÕES COMO : PARA ONDE QUERO IR QUE SERIAM OS HOST (HOSPEDAGEM) , O USER-AGENT OU SEJA SEU NAVEGADOR, ISSO É BOM PARA COMUNICAR O SERVIDOR O TIPO DE CONTEUDO QUE O SEU NAVEGADOR SUPORTA OU FUNCIONALIDADES QUE NÃO FUNCIONA NO SEU NAVEGADOR POIS EXISTEM NAVEGADORES QUE NÃO SUPORTA ALGUMAS FUNCIONALIDADE DE ALGUMA LINGUAGEM , O TAMANHO DOS DADOS QUE ESTÁ ENVIANDO OU OBTENDO, ACCEPT-ENCONDING SERVE PARA INFORMA O SERVIDOR O METODO DE COMPACTAÇÃO DE ARQUIVO SUPORTADO PELO NAVEGADOR, COKKIES USADOS PARA LEMBRAR SUAS INFORMÇÕES COMO: O QUE VOCÊ ACESSOU NO SITE O QUE GOSTOU O QUE QUER E INFORMAÇÕES DE LOGIN COMO USUARIO E SENHa , SÃO USADOS PRINCIPALMENTE PARA EXIBIR ANUCIOS MAIS PARECIDOS COM SEU GOSTO E FACILITAR A INTERAÇÃO E USO DO SITE OBS: ESSES DADOS SÃO ARMAZENADOS NO SEU NAVEGADOR E QUE AO VOCÊ ENTRAR NO SITE ELES SÃO CARREGADOS E ENVIADOS NA SUA SOLICITAÇÃo. NO CABEÇALHO DO SERVIDOR NA RESPOSTA PARA O CLIENTE HÁ INFORMAÇÕES COMO : A INFORMAÇÃO ARAMZENADA NO COKKIE QUE É ENVIADA NOVAMENTE A CAD SOLICITAÇÃO NOVA, CACHE-CONTROL: POR QUANTO TEMPO IRA ARMAZENAR A RESPOSTA NO CAHCHE DO NAVEGADOR ANTES DE AOLICITAR ELA, CONTENT-TYPE: TIPO DE CONTEUDO QUE FOI ENVIADO COMO RESPOSTA, ACCEPT-ENCONDING: O TIPO DE COMPACTAÇÃO USADA PARA ENVIAR O ARQUIVO. 


MAIS SOBRE O PRINCIPAL PROTOCOLO USADO NA INTERNET O TCP/IP:

NELE EXISTEM 4 CAMADAS:

1- CAMADA DE REDE OU CAMADA DE IP: CAMADA ONDE É USADA PARA ENDEREÇAR OS DISPOSIVOS COM NUMEROS DE IP E IDENTIFICAR ELES E A PROPRIA REDE. VEM JUNTO A PARTE DO ENLACE DE DADOS VINDOS DA CAMADA DE TRANSPORTECAMADA DE REDE OU FISICA ONDE RECEBE OS DADOS E ENVIA PARA O DISPOSITIVO, CAMADA DE LAÇO DE DADOS PODE SER FISICA COMO UM CABO ETHERET OU ADPTADOR WIFI DO NOTEBOOK.

3- CAMADA DE TRASNPORTE: ONDE FICA O PROTOCOLO DE CONTROLE DE INTERENET RESPONSAVEL PELO TRAFEGO E CONTROLE DE DADOS PARA MANDAR OS DADOS PARA O DISPOSITIVO NA REDE, EXISTE O PROTOCOLO DE REDE UDP, ELA NÃO É BASEADA EM CONEXÃO COM O DISPOSITIVO EM SI APENAS SE COMUNICA E TRANSFERE ARQUIVOS SEM VERIFICAR A INTEGRIDADE OU SEGURANÇA DELES MAS COM ISSO A TRANFERENCIA E COMUNICAÇÃO É BEM MAIS RAPIDA.

4- CAMADA DE APLICAÇÃO: ONDE O USUARIO INTERAGE COM A REDE DE DIFERENTES FORMA, COMO USANDO UM APLICATIVO OU COMPARTILHANDO ARQUIVO E SE COMUNICANDO COM OUTROS DISPOSITIVOS ATRAVES DE PROTOCOLOS(FORMAS DE SE COMUNICAR, ONDE CADA FORMA TEM REGRAS DIFERENTES)


MODELO OSI: MODELO DE REDE ONDE É UM SISTEMA ABERTO DE INTER CONEXÕES DE COMPUTADORES POR DIVEROS TIPOS DE PROTOCOLOS

PROCESSO DE ESTABELCIMENTO DE COMUNICAÇÃO ATRAVES DO PROTOCOLO TCP(CONTROL PROTOCOL TRANSMITION)

1 - SYN: O DISPOSITIVO ENVIA UM PACOTE SYN(SINCRONIZE) PEDINDO UMA SINCRONIZAÇÃO DO O DISPOSITIVO DE DESTINO

2 - SYN-ACK: NO SYN-ACK(SYNCHRONIZE-ACKNOWLEDGE) SINCRONIZANDO-RECONHECENDO, OCORRE QUANDO O SERVIDOR RECONHECE
O PACOTE SYN E TENTA SE CONECTAR OU ESTABELECER UMA CONEXÃO.

3 - ACK: SE O PACOTE FOR RECONHECIDO E ACEITO O SERVIDOR CONSEGUIR ESTABELECER A CONEXÃO COM O CLIENTE O SERVDOR
ENVIA UM ACK(ACKNOWLEDGE) RECONHECIDO. E OS SISTEMAS ESTABELECEM A CONEXÃO OU RECEBIMENTO DE DADOS DO SERVIDOR
QUE POSTERIORENTE É ENVIADA UMA RESPOSTA PARA O SERVIDOR.

ERROS EM COMUNICAÇÕES VIA PROTOCOLO TCP:

SE O CLIENTE DER UM SYN PARA SE CONECTAR AO SERVIDOR E A TENTATIVA NÃO FOR BEM SUCEDIDA:
PODEM SURGIR ERROS E SINAIS AO NA ANALIZE DO TRAFEGO DE REDE COMO:

TEMPO PERDIDO OU TIMEOUT: QUANDO A TENTATIVA DE CONEXÃO DO CLIENTE OU SERVIDOR DEMORA MUITO E NÃO OCORRE A CONEXÃO
PODEM SURGIR ERROS COMO TIMEOUT COM AVISOS

RST(RESET): QUANDO O SERVIDOR DECIDE NÃO ACEITAR A CONEXÃO APOS RECEBER O PACOTE SYN DO CLIENTE ENTÃO ELE ENVIA A RESPOSTA
RST(RESET) PARA RESETAR A CONEXÃO E INDICAR QUE A CONEXÃO NÃO FOI ESTABELECIDA.

ICMP: ATRAVES DO PROTOCOLO ICMP(PROTOCOL MENSSAGE INTERNET) O SERVIDOR PODE ENVIAR PACOTES COM AVISOS DE QUE A CONEXÃO NÃO PODE SER ESTABELECIDA
OU QUE ESTÁ INCESSIVEL PARA O CLIENTE: RESPOSTA PARA O CLIENTE: "Destination Unreachable" ou "Destino inacessivel".

PROXY: PROXY EM PORTUGUÊS PROUCURADOR, É UM SERVIDOR INTERMÉDIARIO ENTRE A REQUISIÇÃO DO CLIENTE
PARA O SERVIDOR DESTINO. O CLIENTE FAZ UMA REQUISIÇÃO E O PROXY CAPTURA ELA E A PARTIR DAI O PROXY É QUE FAZ 
A REQUISIÇÃO PARA O SERVIDOR DESTINO, E QUE AO OBTER A RESPOSTA DO SERVIDOR ENTÃO O PROXY ENVIA PARA O COMPUTADOR REAL QUE FEZ A SOLICITAÇÃO, O SERVIDOR PROXY ASSIM COMO QUALQUER OUTRO SERVIDOR POSSUI ENDEREÇOS
IP DIFERENTES ASSIM PODENDO DIFICULTAR O RASTREAMENTO REAL DE ONDE VEIO A VERDADEIRA SOLICITAÇÃO. O PROXY
TAMBEM PODE SER UTILIZADA PARA FILTRAR E ADMINISTRAR O TRAFEGO DA REDE PODENDO BLOQUEAR CONEXÕES CRIAR REGRAS
NA REDE E INTERCEPTAR O TRAFEGO PARA ACHAR PROBLEMAS OU ATIVIDADES ESTRANHAS ASSIM PROTEJENDO A REDE DE COMPUTADORES
DE UM LUGAR.

FIREWALL:  O FIREWALL SERVE PARA PROTEGER O COMPUTADOR DE REDES CONEXÕES EXTERNAS NÃO AUTORIZADAS QUE PODEM 
SER MALICIOSAS OU DESCONHECIDAS, ASSIM COMO O PROXY ELE FILTRA E IMPEDE CONEXÕES DE ACORDO AS REGRAS QUE FORAM
DEFINIDAS PARA ELE, ASSIM ELE AGE COMO UMA BARREIRA DE PROTEÇÃO CONTRA TROCA DE DADOS COM COMPUTADORES EXTERNOS
NÃO AUTORIZADOS BLOQUEANDO PORTAS DO COMPUTADOR ASSIM OS DISPOSITIVOS EXTERNOS NÃO PODEM SE COMUNICAR COM SEU COMPUTADOR
EM UMA DETERMINADA PORTA POIS ELÁ ESTÁ PROTEGIDA E FECHADA PARA CONEXÕES EXTERNAS POR CONTA DO FIREWALL EM PORTUGUES
BARREIRA DE FOGO. 

O CIDR É UMA NOTAÇÃO USADA PARA INFORMA O PADRÃO DO ENDEREÇO IP OU SEJA PARA PODE DIFERENCIAR ELE
NA REDE E SABER EM QUE CLASSE ESTÁ E QUANTOS HOSTS A NA SUBREDE DELE
